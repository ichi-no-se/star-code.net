## はじめに

アクセス解析を置きました．ちょっと気になったので．

当 Web サイトを開設して 1 年 3 ヶ月，一切アクセスを解析していませんでした．
アクセスみたところで広告を置いているわけでもなし，アクセスが多くなるように狙って作ろうとかそういう気持ちも一切なく，端的に言うと興味がなかったし面倒だったのですね．

でも，インターネットの片隅にろくに宣伝もせずただ置いてあるだけの Web サイトにどれだけアクセスがあるのか，少し気になったので置いて見ることにしました．

Bot や クローラの類は四六時中来ているのは知っています．一度[ハッキングされました](/blog/hacked)し，`/var/log/nginx/access.log` を見ると大量のアクセスがわかります．

アクセスは大量なのですが，生身の人間がどれだけ見ているかはしっかり見ないとわかりません．
`/var/log/nginx/access.log` を頑張って解析しても良いのですが，それはそれで面倒なので，もう少し簡単な方法を取ることにしました．

## 技術要素

[Umami](https://umami.is)（[GitHub](https://github.com/umami-software/umami)）を使用しています．

デフォルトだとポートが 3000 で立ち上がるのですが，この Web サイト本体とかち合うのでずらしたり，それに伴うルーティングで少し手こずったりしましたが，いい感じに動いているはずです．

`pm2` で自動起動するようにしていますが，ポートを指定するためにはこれの手前に環境変数を入れる必要があるのですね．
以下のように起動するといい感じに動きました（ポート番号 4000 で実行する場合）．

`PORT=4000 pm2 start npm --name "umami" -- start`


HTML の要素を見ると，`<script src="https://analytics.star-code.net/script.js" ...></script>` というのが見られるはずです．
それがアクセス解析用のスクリプトです．

[`https://analytics.star-code.net`](https://analytics.star-code.net) にアクセスするとアクセス解析の画面になりますが，皆さんにパスワードを共有することはありません．悪しからず．

## おわりに

Google Analytics を使おうかとも思いましたが，セルフホストした方が綺麗だしデータを Google 側に渡さなくて済むしと思い，このような形にしてみました．

予想すると [3D Gaussian Splatting 用の Docker 環境を作った](/blog/gaussian-splatting-docker)のアクセス数が一番多いんじゃないかなと思ったり．
